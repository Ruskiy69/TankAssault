void asset::CSound2D::Move ( const math::CVector2 &  Rate)

Because OpenAL automatically handles sound volume based on location to the listener, it may sometimes be necessary to move the sound source closer or farther to the listener. This moves it at a certain rate, not to a certain location.

bool CSound2D::Play ( )

Due to the somewhat complicated source-management system in place, buffers are bound to an OpenAL source on demand. So whenever this method is called, an available source is found, and then the buffer is bound to it, and the audio is played. The CAssetManager is responsible for unloading the source after playback has completed in full.

Returns:
TRUE if the sound played, FALSE if not.

bool CSound2D::UnloadSource ( )

Since OpenAL is limited to 256 audio sources, it is important to be conservative with the available resources. Though I doubt all 256 will ever be used simultaneously, a system is in place to minimize source-hogging. This method should only be called by an CAssetManager after checking that playback has been completed.

Returns:
TRUE if the source was unloaded successfully, FALSE if not, or no source loaded.
